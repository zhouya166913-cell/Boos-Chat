# 企业管理助手四项目启动说明

本文档说明当前四个项目如何启动：

| 项目 | 目录 | 技术栈 | 端口 |
| --- | --- | --- | --- |
| 后端服务 | `boss-chat-server` | Java Spring Boot + MySQL | `9090` |
| 管理系统 | `boss-chat-web` | Vue 3 + TypeScript + Element Plus | `9000` |
| App | `boss-chat-app` | Flutter + Dart | `9999`（仅 Flutter Web 调试） |
| 小程序 | `boss-chat-miniapp` | Vue 3 + uni-app | `9900`（仅 H5 预览） |

> `9999` 和 `9900` 都只是本地调试预览端口。原生 App 与真正运行在微信 / 支付宝里的小程序，本身并不会“启动在某个 HTTP 端口上”。

当前阶段已经从测试表切换到第一版正式认证模型，后端默认提供：

- 本地账号密码登录
- RBAC 基础表结构
- 微信 / QQ 等第三方账号绑定的扩展位
- 智能体、会话、消息、用量记录基础结构
- Swagger UI

如果你想按“每个项目对应一个开发工具”的方式启动，见：

```text
四项目编辑器启动说明.md
```

## 二、数据库准备

### 1. 确认 MySQL 已启动

Windows 下可先确认 `MySQL80` 服务已经运行。

### 2. 创建开发数据库

数据库初始化脚本位于：

```text
boss-chat-server/database/create_database.sql
```

脚本会创建：

```text
boss_chat_dev
```

当前本机开发环境默认连接配置位于：

```text
boss-chat-server/src/main/resources/application.yml
```

默认值为：

```text
host=localhost
port=3306
database=boss_chat_dev
username=root
password=请填写你的数据库密码
```

### 3. 数据库迁移说明

后端第一次启动时，Flyway 会自动执行：

```text
boss-chat-server/src/main/resources/db/migration/
```

当前正式迁移脚本：

```text
V1__create_system_auth_tables.sql
V2__real_project_phase1.sql
```

启动成功后，数据库中会自动出现：

- `sys_user`
- `sys_role`
- `sys_permission`
- `sys_user_role`
- `sys_role_permission`
- `sys_oauth_account`
- `sys_login_record`
- `ai_agent`
- `ai_conversation`
- `ai_message`
- `ai_usage_record`
- `flyway_schema_history`

## 三、启动后端服务

### 方式 A：在 IDEA 中启动

1. 用 IDEA 打开目录：

```text
boss-chat-server
```

2. 等待 Maven 依赖导入完成。  
3. 运行启动类：

```text
com.zhiyinhui.bosschat.BossChatApplication
```

4. 控制台出现以下内容，说明启动成功：

```text
Tomcat started on port 9090
Started BossChatApplication
```

### 方式 B：使用 Maven 启动

在后端目录执行：

```powershell
cd D:\LanTu\life-cycle-management-system\bossChat\boss-chat-server
mvn spring-boot:run
```

当前智能体默认配置：

```text
provider=zhipu
baseUrl=https://open.bigmodel.cn/api/paas/v4
model=glm-4-flash-250414
```

当前开发机支持在后端项目根目录放置本地文件：

```text
application-local.yml
```

项目会自动读取该文件；它已加入 `.gitignore`，适合开发阶段保存本机专用 Key。  
如果本地文件不存在，仍可继续使用环境变量 `LLM_API_KEY`。

如果以后需要把个人 Key 换成公司 Key，见：

```text
智谱模型配置与API Key更换说明.md
```

### 后端地址

```text
http://localhost:9090
```

当前登录接口：

```text
POST http://localhost:9090/api/auth/login
```

Swagger UI：

```text
http://localhost:9090/swagger-ui.html
```

> 注意：Swagger 不在 `/api` 前缀下。  
> 正确地址是 `http://localhost:9090/swagger-ui.html`，不是 `http://localhost:9090/api/swagger-ui/...`。

Swagger 登录后调试受保护接口的方式：

1. 先调用 `POST /api/auth/login`
2. 复制返回结果中的 `token`
3. 点击 Swagger 右上角 `Authorize`
4. 在 `Sa-Token` 中直接粘贴 token
5. 再调用需要登录的接口，例如 `GET /api/auth/me`

> 当前项目使用的请求头名称是 `satoken`，不是 `Authorization: Bearer ...`。

## 四、启动管理系统

```powershell
cd D:\LanTu\life-cycle-management-system\bossChat\boss-chat-web
npm install
npm run dev
```

默认访问：

```text
http://localhost:9000
```

管理系统已配置：

```text
/api -> http://localhost:9090
```

当前管理系统主要页面：

```text
/dashboard   仪表盘模板页
/chat        智能体对话页
/users       用户管理
/agents      智能体管理
```

如果需要让同一局域网中的其他设备访问管理系统：

1. 前端 Vite 需要监听所有网卡
2. 后端 CORS 需要放行对应来源模式

当前开发环境已支持：

```text
http://192.168.*.*:9000
```

## 五、启动 Flutter App

当前机器已经安装 Flutter 3.41.9、Android Studio 和 Android SDK，`boss-chat-app` 也已经补齐为正式 Flutter 工程。

第一次进入项目或依赖变化后先执行：

```powershell
cd D:\LanTu\life-cycle-management-system\bossChat\boss-chat-app
flutter pub get
```

### Flutter Web 调试

```powershell
flutter run -d chrome --web-port=9999 --dart-define=API_BASE_URL=http://localhost:9090/api
```

默认访问：

```text
http://localhost:9999
```

### Android / iOS 调试

- iOS 模拟器可继续使用 `http://localhost:9090/api`
- Android 模拟器通常应使用：

```powershell
flutter run --dart-define=API_BASE_URL=http://10.0.2.2:9090/api
```

- 真机调试时，需要改成电脑在局域网内可访问的地址

安卓启动、模拟器创建、设备选择与联调细节，单独见：

```text
Flutter App 安卓启动与联调说明.md
```

### Android Studio 启动提示

如果在 Android Studio 中打开 `lib/main.dart` 后，右上角仍然显示 `Add Configuration`，且运行按钮是灰色：

1. 先确认已安装并启用 Flutter / Dart 插件
2. 确认 Flutter SDK 指向：

```text
D:\flutter\flutter_windows_3.41.9-stable\flutter
```

3. 如仍未自动生成配置，可手动新建 Flutter 运行配置，入口文件选择：

```text
lib/main.dart
```
