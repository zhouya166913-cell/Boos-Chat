# 四项目编辑器启动说明

本文档专门说明：当前四个项目分别放在哪个开发工具中，以及在对应工具里应该如何启动。

| 项目 | 推荐开发工具 | 说明 |
| --- | --- | --- |
| 后端服务 `boss-chat-server` | IntelliJ IDEA | 直接运行 Spring Boot |
| 管理系统 `boss-chat-web` | Visual Studio Code | 在集成终端运行 Vite |
| App `boss-chat-app` | Android Studio | 直接运行 Flutter App |
| 小程序 `boss-chat-miniapp` | 微信开发者工具 | 打开编译后的微信小程序产物 |

> 推荐启动顺序：  
> 1. 先启动 MySQL  
> 2. 再启动后端  
> 3. 最后启动要联调的前端项目

---

## 一、后端服务：在 IntelliJ IDEA 中启动

### 1. 打开项目

在 IDEA 中打开：

```text
D:\LanTu\life-cycle-management-system\bossChat\boss-chat-server
```

等待右下角 Maven 导入完成。

### 2. 推荐方式：直接运行启动类

打开：

```text
src/main/java/com/zhiyinhui/bosschat/BossChatApplication.java
```

在 `main` 方法左侧点击绿色运行按钮，或右键选择运行：

```text
Run 'BossChatApplication'
```

IDEA 会为 Spring Boot 自动创建运行配置。

### 3. 你也可以用 Maven 面板启动

如果你更习惯 Maven 面板，也可以在 IDEA 右侧 Maven 工具窗口中执行：

```text
spring-boot:run
```

本质上等价于命令行：

```powershell
mvn spring-boot:run
```

### 4. 启动成功标志

控制台出现：

```text
Tomcat started on port 9090
Started BossChatApplication
```

说明后端启动成功。

### 5. 启动后可访问

```text
后端地址：http://localhost:9090
Swagger：http://localhost:9090/swagger-ui.html
```

## 二、管理系统：在 VS Code 中启动

### 1. 打开项目

在 VS Code 中打开文件夹：

```text
D:\LanTu\life-cycle-management-system\bossChat\boss-chat-web
```

### 2. 打开集成终端

在 VS Code 中打开终端：

```text
Terminal -> New Terminal
```

### 3. 第一次启动时先安装依赖

```powershell
npm install
```

### 4. 启动管理系统

```powershell
npm run dev
```

### 5. 启动成功标志

终端中会看到 Vite 输出本地地址，浏览器访问：

```text
http://localhost:9000
```

### 6. 当前项目说明

管理系统已经把：

```text
/api
```

代理到：

```text
http://localhost:9090
```

所以只要后端已经启动，登录页就可以直接联调。

## 三、App：在 Android Studio 中启动

### 1. 打开项目

在 Android Studio 中打开：

```text
D:\LanTu\life-cycle-management-system\bossChat\boss-chat-app
```

如果 Android Studio 提示安装 Flutter / Dart 插件，先安装并重启 IDE。

### 2. 第一次进入项目时

可以在 Android Studio 底部 `Terminal` 中执行：

```powershell
flutter pub get
```

### 3. 选择运行设备

在顶部工具栏选择：

- 已连接 Android 真机
- 或 Android Emulator
- 如果只想临时调试 Web，也可以选择 Chrome

如果你当前还没有 Android 模拟器，或设备下拉框里只看到 `Windows / Chrome / Edge`，请先看独立文档：

```text
Flutter App 安卓启动与联调说明.md
```

### 4. 运行 App

打开：

```text
lib/main.dart
```

点击 Android Studio 顶部运行按钮即可启动。

如果右上角仍然显示：

```text
Add Configuration
```

并且运行按钮是灰色，说明 Android Studio 还没有为当前项目生成 Flutter 运行配置。此时按下面处理：

1. 确认已安装并启用 Flutter / Dart 插件
2. 打开 `File -> Settings -> Languages & Frameworks -> Flutter`
3. 确认 Flutter SDK 路径为：

```text
D:\flutter\flutter_windows_3.41.9-stable\flutter
```

4. 如果仍然没有自动生成配置，点击右上角 `Add Configuration`，手动新建 `Flutter` 类型配置：

```text
Name: boss-chat-app
Dart entrypoint: lib/main.dart
Working directory: D:\LanTu\life-cycle-management-system\bossChat\boss-chat-app
```

### 5. Android 模拟器联调本机后端

如果你用的是 Android 模拟器，本项目建议让 App 访问：

```text
http://10.0.2.2:9090/api
```

可以在 Android Studio 中通过运行配置传入：

```text
--dart-define=API_BASE_URL=http://10.0.2.2:9090/api
```

如果你是直接在 Android Studio 终端启动，也可以执行：

```powershell
flutter run --dart-define=API_BASE_URL=http://10.0.2.2:9090/api
```

> 更完整的 Android Studio 设备选择、模拟器创建、`Restart Flutter Daemon`、`Dart SDK is not configured` 等问题处理，统一见 `Flutter App 安卓启动与联调说明.md`。

### 6. Flutter Web 调试

如果你想在浏览器里调试 App，可以在 Android Studio 底部终端执行：

```powershell
flutter run -d chrome --web-port=9999 --dart-define=API_BASE_URL=http://localhost:9090/api
```

浏览器访问：

```text
http://localhost:9999
```

> `9999` 只是 Flutter Web 调试端口。真正的 Android / iOS App 本身没有固定 HTTP 端口。

## 四、小程序：在微信开发者工具中启动

### 先说明一个关键点

`boss-chat-miniapp` 是 **uni-app 源码项目**。  
它**当然可以在微信开发者工具中启动、预览和调试**，只是微信开发者工具打开的不是 `src` 源码根目录，而是 **uni-app 编译后的微信小程序产物目录**。

也就是说：

- 源码目录：

```text
D:\LanTu\life-cycle-management-system\bossChat\boss-chat-miniapp
```

- 微信开发者工具实际导入目录：

```text
D:\LanTu\life-cycle-management-system\bossChat\boss-chat-miniapp\dist\dev\mp-weixin
```

### 1. 先生成微信小程序产物

在任意终端中进入：

```powershell
cd D:\LanTu\life-cycle-management-system\bossChat\boss-chat-miniapp
```

第一次先安装依赖：

```powershell
npm install
```

启动微信小程序开发构建：

```powershell
npm run dev:mp-weixin
```

为了方便日常开发，当前项目也提供了一个默认命令：

```powershell
npm run dev
```

它等价于：

```powershell
npm run dev:mp-weixin
```

这个命令需要保持运行。

### 2. 在微信开发者工具中导入

打开微信开发者工具，选择：

```text
导入项目
```

项目目录选择：

```text
D:\LanTu\life-cycle-management-system\bossChat\boss-chat-miniapp\dist\dev\mp-weixin
```

之后就可以在微信开发者工具里正常：

- 启动小程序
- 预览页面
- 调试代码
- 查看控制台
- 查看网络请求
- 做真机预览

### 3. 修改代码后的工作方式

日常开发时建议这样配合：

1. 在源码目录里改 `src` 下的代码
2. 保持 `npm run dev` 或 `npm run dev:mp-weixin` 运行
3. 编译产物自动更新到 `dist/dev/mp-weixin`
4. 微信开发者工具读取最新产物进行预览和调试

### 4. 注意 `9900` 的含义

```text
http://localhost:9900
```

只是小程序项目的 **H5 预览地址**。  
如果你现在是在微信开发者工具里联调微信小程序，就不需要打开 `9900`。

## 六、推荐你平时这样启动

### 如果你只开发管理系统

1. IDEA 启动后端  
2. VS Code 启动管理系统  

### 如果你只开发 App

1. IDEA 启动后端  
2. Android Studio 启动 App  

### 如果你只开发微信小程序

1. IDEA 启动后端  
2. 终端执行 `npm run dev` 或 `npm run dev:mp-weixin`  
3. 微信开发者工具导入 `dist/dev/mp-weixin`  

### 如果你要四端一起联调

1. MySQL  
2. IDEA：后端  
3. VS Code：管理系统  
4. Android Studio：App  
5. 微信开发者工具：小程序
