## 当前阶段目标

- 保持后端服务独立
- 让管理系统、App、小程序共用同一套后端接口
- 建立脚本化数据库迁移机制
- 已完成第一版正式认证与权限骨架
- 四个主线项目的基础模板均已实际跑通

## 最终技术栈

| 端 | 技术栈 | 当前状态 |
| --- | --- | --- |
| 后端服务 | Java Spring Boot + MySQL | 已建立正式认证骨架 |
| 前端管理系统 | Vue 3 + TypeScript + Element Plus | 已建立后台页面骨架 |
| App | Flutter + Dart | 已补齐为正式工程 |
| 小程序 | Vue 3 + uni-app | 已建立登录模板 |

## 端口约定

| 项目 | 端口 | 说明 |
| --- | --- | --- |
| 后端服务 | 9090 | 真正提供接口服务 |
| 管理系统 | 9000 | Vite 开发端口 |
| App | 9999 | Flutter Web 本地调试端口；原生 App 本身没有 HTTP 监听端口 |
| 小程序 | 9900 | uni-app H5 预览端口；真正的小程序运行时不依赖这个端口 |

## 前端之间的关系

三个前端项目彼此独立：

- 管理系统不依赖 App
- App 不依赖小程序
- 小程序也不依赖管理系统

它们共享的只有：

- 同一个后端服务
- 同一套接口约定
- 后续可以逐步复用的业务模型与设计语言

## 当前已经明确的登录与权限方向

- 本地账号密码登录先行
- 权限采用 RBAC
- 权限归属 `sys_user`
- 微信 / QQ 等第三方登录通过 `sys_oauth_account` 绑定到本地用户
- 同一个用户无论使用哪种方式登录，最终都落到同一套角色与权限上
