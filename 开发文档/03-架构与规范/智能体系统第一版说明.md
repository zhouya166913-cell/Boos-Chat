# 智能体系统第一版说明

本文档说明当前系统中“真正智能体”第一版的能力、边界和后续演进方向。

---

## 一、当前定位

当前系统已经从“只能聊天的角色化助手”推进到“可以调用工具的工作区智能体”。

当前已经具备：

- 长期记忆
- 知识库
- 工作流
- 工具调用
- 本地工作区能力
- 工具执行记录
- 工具调用前人工确认
- 前端手动停止 AI 任务

第一版目标是先把这条主链路跑通：

```text
用户提出任务 -> 模型理解任务 -> 选择工具 -> 请求确认 -> 执行本地能力 -> 返回结果 -> 保留上下文
```

---

## 二、入口定位

### 1. AI 对话 `/chat`

这是老板和最终用户体验智能体能力的主要入口。

当前支持：

- 普通大模型对话
- 工具型智能体对话
- 会话记录
- 新建会话 / 继续会话
- 流式输出
- 工作流选择
- 上下文记录展示与清空
- 敏感工具调用确认
- 手动停止当前 AI 任务
- 会话列表标记来源：`大模型` / `智能体`

说明：

- 普通大模型只负责聊天、问答、分析，不执行本地工具。
- 工具型智能体可以结合记忆、知识库、工作流，并调用文件、命令、Git、测试等本地工具。

### 2. 智能体工作台 `/workbench`

这是管理员用于配置和调试智能体能力的后台页面。

当前支持：

- 执行本地工作区任务
- 查看工具调用过程
- 清空当前工作台上下文
- 停止当前 AI 任务
- 维护长期记忆
- 维护知识库
- 维护工作流

工作台右侧“智能体能力配置”分为三个 Tab：

| Tab | 作用 | 当前能力 |
| --- | --- | --- |
| 长期记忆 | 保存当前登录用户的长期偏好或固定背景 | 查看、新增、编辑、删除、启用 / 停用 |
| 知识库 | 保存项目资料、业务方法论、老板需求、企业咨询模板等稳定内容 | 查看、新增、编辑、删除、启用 / 停用 |
| 工作流 | 保存可复用的做事步骤和任务约束 | 查看、新增、编辑、删除、启用 / 停用 |

## 三、后端数据模型

当前核心表：

```text
ai_agent
ai_conversation
ai_message
ai_usage_record
ai_memory
ai_knowledge_document
ai_workflow
ai_tool_execution
```

### 1. 长期记忆 `ai_memory`

用途：保存当前登录用户的长期偏好或固定背景。

特点：

- 记忆归属于用户
- 同一用户下 `memory_key` 唯一
- 启用的记忆会注入智能体上下文
- 当前支持新增、查询、修改、删除、启用 / 停用

### 2. 知识库 `ai_knowledge_document`

用途：保存稳定的业务资料、项目资料、方法论、老板需求等内容。

特点：

- 当前是轻量文本知识库，不是最终向量检索版
- 智能体执行任务时会按任务内容选择部分相关知识注入上下文
- 当前支持新增、查询、修改、删除、启用 / 停用

### 3. 工作流 `ai_workflow`

用途：保存一套可复用的任务执行步骤。

特点：

- `workflow_code` 唯一
- 执行任务时可选择工作流
- 智能体会把工作流作为任务约束参考
- 当前支持新增、查询、修改、删除、启用 / 停用

当前默认预置：

```text
coding_task
```

### 4. 工具执行记录 `ai_tool_execution`

用途：记录智能体执行工具的过程，方便审计和排查。

## 四、当前已实现工具

| 工具 | 作用 |
| --- | --- |
| `list_directory` | 查看目录内容 |
| `read_file` | 读取普通文本文件 |
| `read_excel` | 读取 `.xlsx` Excel 工作簿内容 |
| `create_file` | 新建文件 |
| `write_file` | 覆盖写入文件 |
| `search_text` | 在目录中搜索文本 |
| `run_command` | 执行 PowerShell 命令 |
| `git_status` | 查看 Git 状态 |
| `git_diff` | 查看 Git 差异 |
| `run_tests` | 在指定目录运行测试命令 |

注意：

- `.xlsx` 文件不要用 `read_file` 当文本读，应使用 `read_excel`。
- `.xls` 当前暂不支持，建议另存为 `.xlsx`。
- Excel 日期、格式等高级内容目前可能以原始值形式展示，后续可继续增强。

## 五、本地工作区配置

当前配置位于后端配置项：

```yaml
app:
  agent:
    workspace-root: ${AGENT_WORKSPACE_ROOT:..}
    allowed-roots: ${AGENT_ALLOWED_ROOTS:}
    command-timeout-seconds: ${AGENT_COMMAND_TIMEOUT_SECONDS:120}
    max-read-characters: ${AGENT_MAX_READ_CHARACTERS:40000}
    max-tool-rounds: ${AGENT_MAX_TOOL_ROUNDS:30}
```

含义：

- `workspace-root`：默认工作目录，普通相对路径基于它解析。
- `allowed-roots`：额外授权目录，支持英文逗号或分号分隔多个目录。
- `command-timeout-seconds`：命令执行超时时间。
- `max-read-characters`：单次读取文件最大字符数。
- `max-tool-rounds`：单轮任务最多工具调用轮次，避免无限循环。

示例：

```yaml
app:
  agent:
    workspace-root: D:/LanTu/life-cycle-management-system/bossChat
    allowed-roots: D:/LanTu;C:/Users/Lenovo/Desktop
```

如果确实要让智能体访问整盘，可以配置：

```yaml
app:
  agent:
    allowed-roots: C:/;D:/
```

但这是高风险配置，只建议开发者本机临时使用，不建议客户环境默认开启。

## 六、安全边界

当前已经做了以下控制：

- 默认限制在工作区根目录内
- 支持额外授权目录白名单
- 未授权目录访问需要用户确认
- 新建文件、覆盖文件、执行命令、运行测试前需要用户确认
- `run_command` 始终视为敏感工具
- 用户确认只放行本次工具调用，不会永久加入授权目录
- 工具型智能体接口当前要求超级管理员权限
- 工具执行过程落库
- 前端支持停止当前 AI 任务

仍建议后续补充：

- 文件修改 diff 预览
- 文件修改回滚
- 命令白名单 / 黑名单
- 更细粒度权限控制
- 更完整审计日志

## 七、上下文能力

当前上下文分两类：

1. AI 对话上下文：基于数据库中的会话和消息记录。
2. 工作台上下文：当前前端保存在浏览器本地，用于连续执行工作台任务。

工作台上下文当前特点：

- 前端保存最近若干条上下文消息
- 刷新页面后仍保留在当前浏览器
- 每次执行任务时会一起传给后端
- 页面支持手动清空上下文

注意：

- 工作台上下文目前不是数据库级多设备会话。
- 如果后续要做多人协作或多设备同步，需要新增工作台会话表和消息表。

## 十、当前结论

当前系统已经具备真正智能体的第一版骨架：

```text
记忆 + 知识库 + 工作流 + 工具 + 本地能力
```
