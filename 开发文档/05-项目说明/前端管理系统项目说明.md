# 前端管理系统项目说明

当前项目对应四项目架构中的**前端管理系统**；它与 Flutter App、小程序项目彼此独立，只共用同一个后端服务。

技术栈：

- Vue 3
- TypeScript
- Vite
- Pinia
- Element Plus

---

## 一、当前已实现页面

| 路由 | 页面 | 当前定位 |
| --- | --- | --- |
| `/login` | 登录页 | 用户名密码登录 |
| `/dashboard` | 仪表盘 | 当前保留写死概览模板，后续接真实统计 |
| `/chat` | AI 对话 | 老板和用户体验智能体能力的主入口 |
| `/checkins` | 签到列表 | 创建课程期数、维护学员、生成每期问卷二维码、查看数据看板 |
| `/survey-records` | 调查记录 | 查看客户问卷记录，支持按课程期数筛选与看板分析 |
| `/users` | 用户管理 | 展示真实用户列表 |
| `/agents` | 智能体管理 | 维护智能体基础配置 |
| `/workbench` | 智能体工作台 | 管理员执行本地任务、维护记忆 / 知识库 / 工作流 |

## 二、AI 对话页当前能力

`AI 对话` 是当前真实测试老板需求的主入口。

当前支持：

- 选择普通大模型或工具型智能体
- 新建会话
- 历史会话列表
- 继续旧会话
- 消息流式输出
- Enter 发送，Shift + Enter 换行
- 手动停止当前 AI 输出
- 上下文记录展示
- 清空当前上下文
- 工作流选择
- 敏感工具调用确认
- 会话列表展示来源标签：`大模型` / `智能体`

说明：

- 普通大模型只负责问答、聊天、分析。
- 工具型智能体可以调用本地工具，例如读文件、读 Excel、搜索目录、执行命令、Git、跑测试等。

## 三、智能体工作台当前能力

`智能体工作台` 现在不再只是临时测试页，而是管理员后台配置页。

左侧用于执行本地工作区任务：

- 选择工具型智能体
- 选择工作流
- 输入任务
- 查看工具执行过程
- 停止当前 AI 任务
- 清空工作台上下文

右侧为“智能体能力配置”，分为三个 Tab：

| Tab | 当前能力 |
| --- | --- |
| 长期记忆 | 查看、新增、编辑、删除、启用 / 停用 |
| 知识库 | 查看、新增、编辑、删除、启用 / 停用 |
| 工作流 | 查看、新增、编辑、删除、启用 / 停用 |

这三类内容会被后端注入到工具型智能体上下文中，属于正式智能体能力，不再视为测试数据。

## 四、当前目录整理方向

管理系统已经从“简单平铺目录”向“业务模块优先”演进：

```text
src/
├─ api/                  # 后端接口封装
├─ layouts/              # 管理端布局
├─ modules/              # 业务模块内部组件、组合式逻辑、样式
│  ├─ auth/
│  └─ chat/
├─ router/               # 路由
├─ stores/               # Pinia 状态
├─ styles/               # 全局样式、页面通用样式
└─ views/                # 页面入口
```

当前已经落地：

- 登录页逻辑拆到 `modules/auth/composables/`
- 登录页样式拆到 `modules/auth/styles/`
- AI 对话页拆出：
  - `modules/chat/composables/`
  - `modules/chat/components/`
  - `modules/chat/styles/`
- 全局样式拆为：
  - `base.css`
  - `layout.css`
  - `pages.css`

后续新增复杂页面时，优先按业务归属拆分组件、逻辑和样式，避免把大量 TS / CSS / HTML 全部堆在一个 `.vue` 文件中。

## 七、课程期数与问卷运营

`/courses` 是课程交付前的运营入口，旧入口 `/checkins` 会自动跳转到该页面：

- 创建期数，例如“第十三期AI运营操盘手”。
- 每一期自动生成独立问卷链接，链接会带 `phase` 参数。
- 二维码默认由前端根据问卷链接生成；如果图床已有自定义二维码，可在期数表单填写二维码图片 URL 覆盖。
- 在期数内先创建分组，每个分组可以手动填写组长。
- 在选中的分组内新增学员，字段为座位号、姓名、备注、手机号、身份证号、是否新学员。
- 公开问卷进入前要求学员填写姓名、手机号、身份证号和新老学员类型，但只校验姓名是否在当前期数学员名单中。
- 学员通过姓名校验进入问卷页后即完成签到，学员名单会显示已签到，并累计进入问卷次数。
- 学员公开填写的手机号、身份证号和新老学员类型会同步回后台学员资料，也会随问卷提交保存。
- 同一学员可以重复填写问卷；系统保留每一次历史提交，但课程管理、调查记录列表和数据看板默认只展示该学员最新一次提交。
- 调查记录列表支持单条删除和当前期数全部删除，删除前会二次确认。
- 数据看板会按本期问卷记录统计学员提交情况、高频痛点和课程思路。
- 数据看板的课程分析支持选择一个已启用智能体模型，点击后会生成本次课程分析，并在抽屉内展示历史分析结果。

`/survey-records` 仍然负责查看问卷原始记录和 AI 诊断详情，但已经支持选择期数，只看某一期的提交记录；该页同样支持当前期数记录删除、数据看板、模型选择课程分析和历史分析查看。

## 七、2026-05-20 AI 对话场景化更新

本次管理系统新增 `/scenes` 场景管理页面，并升级 `/chat` AI 对话页面。

### 页面变化

| 路由 | 页面 | 作用 |
| --- | --- | --- |
| `/chat` | AI 对话 | 选择场景、AI 和工作流后进行真实对话 |
| `/scenes` | 场景管理 | 创建单聊 / 团队场景，并配置场景内可用 AI |

### AI 对话规则

- 顶部先选择场景。
- 场景切换后，只展示该场景下的 AI。
- 单聊场景：切换 AI 会进入该 AI 的独立上下文。
- 团队场景：切换 AI 不清空消息，只决定下一轮由哪个 AI 回复。
- 如果当前 AI 开启图片生成能力，页面会提示“当前 AI 可生成图片”。
- 会话列表支持修改标题和软删除。

## 2026-05-28 本地联调更新

前端开发时不要在 `src/api/http.ts`、`src/api/chat.ts`、`src/api/workbench.ts` 中来回注释接口地址。代码统一请求 `/api`，由 Vite 代理决定转发到本地后端还是服务器后端。

默认开发配置在 `boss-chat-web/.env.development`：

```env
VITE_API_BASE_URL=/api

# Local backend
VITE_API_PROXY_TARGET=http://localhost:9090

# Server backend
# VITE_API_PROXY_TARGET=http://8.162.26.228
```

切换 `VITE_API_PROXY_TARGET` 后需要重启 `npm run dev`。本机私有覆盖配置可以写入 `.env.development.local`，该文件不会提交到 GitHub。
