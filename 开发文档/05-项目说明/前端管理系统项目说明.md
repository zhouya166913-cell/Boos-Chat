# 前端管理系统项目说明

当前项目对应四项目架构中的**前端管理系统**；它与 Flutter App、小程序项目彼此独立，只共用同一个后端服务。

技术栈：

- Vue 3
- TypeScript
- Vite
- Pinia
- Element Plus

---

## 一、当前已实现页面

| 路由 | 页面 | 当前定位 |
| --- | --- | --- |
| `/login` | 登录页 | 用户名密码登录 |
| `/dashboard` | 仪表盘 | 当前保留写死概览模板，后续接真实统计 |
| `/chat` | AI 对话 | 老板和用户体验智能体能力的主入口 |
| `/users` | 用户管理 | 展示真实用户列表 |
| `/agents` | 智能体管理 | 维护智能体基础配置 |
| `/workbench` | 智能体工作台 | 管理员执行本地任务、维护记忆 / 知识库 / 工作流 |

---

## 二、AI 对话页当前能力

`AI 对话` 是当前真实测试老板需求的主入口。

当前支持：

- 选择普通大模型或工具型智能体
- 新建会话
- 历史会话列表
- 继续旧会话
- 消息流式输出
- Enter 发送，Shift + Enter 换行
- 手动停止当前 AI 输出
- 上下文记录展示
- 清空当前上下文
- 工作流选择
- 敏感工具调用确认
- 会话列表展示来源标签：`大模型` / `智能体`

说明：

- 普通大模型只负责问答、聊天、分析。
- 工具型智能体可以调用本地工具，例如读文件、读 Excel、搜索目录、执行命令、Git、跑测试等。

---

## 三、智能体工作台当前能力

`智能体工作台` 现在不再只是临时测试页，而是管理员后台配置页。

左侧用于执行本地工作区任务：

- 选择工具型智能体
- 选择工作流
- 输入任务
- 查看工具执行过程
- 停止当前 AI 任务
- 清空工作台上下文

右侧为“智能体能力配置”，分为三个 Tab：

| Tab | 当前能力 |
| --- | --- |
| 长期记忆 | 查看、新增、编辑、删除、启用 / 停用 |
| 知识库 | 查看、新增、编辑、删除、启用 / 停用 |
| 工作流 | 查看、新增、编辑、删除、启用 / 停用 |

这三类内容会被后端注入到工具型智能体上下文中，属于正式智能体能力，不再视为测试数据。

## 四、当前目录整理方向

管理系统已经从“简单平铺目录”向“业务模块优先”演进：

```text
src/
├─ api/                  # 后端接口封装
├─ layouts/              # 管理端布局
├─ modules/              # 业务模块内部组件、组合式逻辑、样式
│  ├─ auth/
│  └─ chat/
├─ router/               # 路由
├─ stores/               # Pinia 状态
├─ styles/               # 全局样式、页面通用样式
└─ views/                # 页面入口
```

当前已经落地：

- 登录页逻辑拆到 `modules/auth/composables/`
- 登录页样式拆到 `modules/auth/styles/`
- AI 对话页拆出：
  - `modules/chat/composables/`
  - `modules/chat/components/`
  - `modules/chat/styles/`
- 全局样式拆为：
  - `base.css`
  - `layout.css`
  - `pages.css`

后续新增复杂页面时，优先按业务归属拆分组件、逻辑和样式，避免把大量 TS / CSS / HTML 全部堆在一个 `.vue` 文件中。

## 五、启动方式

```powershell
npm install
npm run dev
```

默认开发地址：

```text
http://localhost:9000
```

局域网访问时，需要 Vite 监听 `0.0.0.0`，并确保后端 CORS / 安全配置允许对应来源。

## 七、2026-05-20 AI 对话场景化更新

本次管理系统新增 `/scenes` 场景管理页面，并升级 `/chat` AI 对话页面。

### 页面变化

| 路由 | 页面 | 作用 |
| --- | --- | --- |
| `/chat` | AI 对话 | 选择场景、AI 和工作流后进行真实对话 |
| `/scenes` | 场景管理 | 创建单聊 / 团队场景，并配置场景内可用 AI |

### AI 对话规则

- 顶部先选择场景。
- 场景切换后，只展示该场景下的 AI。
- 单聊场景：切换 AI 会进入该 AI 的独立上下文。
- 团队场景：切换 AI 不清空消息，只决定下一轮由哪个 AI 回复。
- 如果当前 AI 开启图片生成能力，页面会提示“当前 AI 可生成图片”。
- 会话列表支持修改标题和软删除。

### 验证

- `npm run build` 已通过。
