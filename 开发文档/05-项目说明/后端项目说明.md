# 后端项目说明

当前项目是四项目架构中的共享后端服务，供管理系统、Flutter App 和 uni-app 小程序共同调用。

技术栈：

```text
Java Spring Boot + MySQL + Flyway + MyBatis-Plus + Sa-Token + Swagger
```

当前阶段目标：

- 建立正式认证与权限基础模型
- 提供账号密码登录闭环
- 为后续微信 / QQ 登录预留绑定结构
- 提供智能体、会话、消息、用量记录与流式回复基础能力
- 提供工作区智能体、记忆、知识库、工作流和工具调用能力
- 提供 Swagger UI 便于查看当前接口

## 二、数据库与迁移

第一次搭建本地环境时，先执行：

```sql
source database/create_database.sql;
```

后续建表、改表、加索引交给 Flyway 自动执行。

迁移脚本位置：

```text
src/main/resources/db/migration
```

当前核心迁移：

```text
V1__create_system_auth_tables.sql
V2__real_project_phase1.sql
V3__agentic_workspace.sql
V30__course_phases_students_and_survey_links.sql
```

`V30` 新增：

- `course_phase`：课程期数，包含期数名称、课程名称、问卷路径、自定义二维码图片 URL、启用状态。
- `course_student`：期数学员，包含姓名、手机号、身份证号、是否新学员。
- `survey_record.phase_id` / `survey_record.student_id`：让问卷记录归档到对应期数和学员。

课程运营接口：

```text
GET    /api/admin/course-phases
POST   /api/admin/course-phases
PUT    /api/admin/course-phases/{phaseId}
GET    /api/admin/course-phases/{phaseId}/students
POST   /api/admin/course-phases/{phaseId}/students
PUT    /api/admin/course-phases/{phaseId}/students/{studentId}
DELETE /api/admin/course-phases/{phaseId}/students/{studentId}
GET    /api/admin/course-phases/{phaseId}/dashboard
```

发布注意事项：
- 本次 `V30__course_phases_students_and_survey_links.sql` 只新增 `course_phase`、`course_student` 两张表，并给 `survey_record` 增加 `phase_id`、`student_id` 两个可空字段。
- 迁移不会删除或清空原有问卷记录；历史记录会保留，只是没有期数归属。
- 发布到服务器后，后端启动时 Flyway 会自动执行 `V30`，执行结果可在 `flyway_schema_history` 中确认。
- 如果某个环境曾经手工加过同名字段或索引，需要先确认表结构，避免重复执行导致迁移失败。

问卷提交接口支持可选 `phaseCode`，来自链接参数 `?phase=...`。如果手机号命中该期数学员，系统会自动绑定 `student_id`。

## 六、认证与权限

当前后端采用：

```text
Sa-Token + 本地账号密码登录 + RBAC
```

当前请求头名称：

```text
satoken
```

不是：

```text
Authorization: Bearer ...
```

默认开发账号：

```text
username: admin
password: Admin@123
```

默认角色：

```text
super_admin
user
```

权限长期决策：

- 权限挂在 `sys_user` 上
- 微信 / QQ 等第三方登录通过 `sys_oauth_account` 绑定到 `sys_user`
- 第三方登录只是身份来源，不承载权限体系

## 八、当前主要接口

```text
POST /api/auth/login      # 登录
```

### 2. AI 对话

```text
GET  /api/chat/scenes                    # 查询可用 AI 场景
GET  /api/chat/agents                    # 查询可用 AI
GET  /api/chat/workflows                 # 查询当前 AI 可用工作流，参数：agentId
GET  /api/chat/conversations             # 查询会话列表
GET  /api/chat/conversations/{id}        # 查询会话详情
POST /api/chat/attachments               # 上传对话附件，支持图片、.xlsx、PDF、Word、文本和视频
POST /api/chat/messages                  # 同步对话，工具型智能体不走该接口
POST /api/chat/messages/stream           # SSE 流式对话，普通大模型与工具型智能体均可使用
POST /api/chat/approvals/{id}/approve    # 同意聊天中的工具调用
POST /api/chat/approvals/{id}/reject     # 拒绝聊天中的工具调用
```

当前注意：

- `GET /api/chat/workflows?agentId=...` 已补齐，聊天页只读取已启用工作流，管理端仍负责工作流增删改。
- 旧日志 `NoResourceFoundException: No static resource api/chat/workflows` 的根因是此前聊天侧缺少该接口；当前已修复。
- 对话附件和 AI 生成/编辑图片当前均落本地 `uploads` 目录，便于测试阶段清理资源。

## 十一、2026-05-21 模型管理动态接口路径

模型管理已经从“固定模型名 + 固定调用路径”调整为“供应商 Base URL + 模型接口路径”的动态组合。

### 数据字段

`ai_model` 新增：

```text
api_path          # 模型接口路径，例如 /chat/completions、/images/generations、/zrag/retrieval/retrieve
billing_type      # 计费标识：free / paid / unknown
official_doc_url  # 官方 API 文档地址
```

迁移文件：

```text
boss-chat-server/src/main/resources/db/migration/V11__dynamic_model_endpoint_config.sql
```

### 管理接口

```text
GET    /api/admin/model-management/models
POST   /api/admin/model-management/models
PUT    /api/admin/model-management/models/{modelId}
DELETE /api/admin/model-management/models/{modelId}
```

删除模型时，后端会先检查模型下是否仍有 API Key、是否仍被智能体引用、是否已有图片生成记录。有关联时会返回冲突错误，避免删除正在使用的模型配置。

### 调用规则

运行时不再在代码里固定拼接 `/chat/completions` 或 `/images/generations`。

```text
最终请求地址 = ai_model_provider.base_url + ai_model.api_path
```

例如：

```text
https://open.bigmodel.cn/api/paas/v4 + /chat/completions
https://open.bigmodel.cn/api/paas/v4 + /images/generations
https://example.com/api + /zrag/retrieval/retrieve
```

相关代码：

```text
LlmChatService              # 对话 / 工具调用按模型 api_path 请求
AiImageGenerationService    # 图片生成按模型 api_path 请求
AiModelManagementService    # 模型字段维护与删除保护
AdminAiModelController      # 模型删除接口
```
